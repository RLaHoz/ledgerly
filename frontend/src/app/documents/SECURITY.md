# Security Policy

## Supported Versions
Active development branch is supported (`develop`), and the latest release branch from `main`.

## Reporting a Vulnerability
Do not open public issues for security vulnerabilities.

Please include:
- vulnerability summary
- impact assessment
- steps to reproduce
- proof of concept (if available)
- suggested remediation

Use private reporting to maintainers. If private channel is unavailable, open a minimal public issue without exploit details and request private contact.

## Response Targets
- Initial triage: within 72 hours
- Risk classification: within 5 business days
- Remediation plan: as soon as severity is confirmed

## Scope
- Authentication and authorization flaws
- Data exposure risks
- Dependency vulnerabilities with practical exploitability
- Sensitive logging/leakage of secrets or tokens
